Git

What is Git?

Git is a free and open source distributed version control system. Git’s purpose is to keep track of projects and files as they change over time with manipulations happening from different users. Git stores information about the project’s progress on a repository. A repository has commits to the project or a set of references to the commits called heads. All this information is stored in the same folder as the project in a sub-folder called .git and will mostly be hidden by default in most systems.

Git Branches

In Git, a branch is a pointer to a specific commit. The branch pointer moves along with each new commit you make, and only diverges in the graph if a commit is made on a common ancestor commit. There are various commands you can take in Git to work with your branches.

Commands:-

1. Git branch yourbranchname   #create your branch
2. Git checkout yourbranchname   #move to your branch

Git Merge

The git merge command lets you take the independent lines of development created by git branch and integrate them into a single branch.

Commands:-

1. Git branch yourbranchname   #create your branch
2. Git checkout yourbranchname   #move to your branch
3. Make your changes!
4. Git status   #check your changes
5. Git add *   #stage your changes
6. Git commit -m “your message here”   #commit your changes
7. Git checkout master   #switch to master branch
8. Git merge yourbranchname
9. Git push   #push changes back to remote

Git Commit

The git commit command captures a snapshot of the project's currently staged changes. Committed snapshots can be thought of as “safe” versions of a project—Git will never change them unless we explicitly ask it to. Prior to the execution of git commit, The git add command is used to promote or 'stage' changes to the project that will be stored in a commit.

Commands:-

1. git commit   #This will launch a text editor for message
2. git commit -m "commit message"   #commit by giving message

Git Tags

Tags are ref's that point to specific points in Git history. Tagging is generally used to capture a point in history that is used for a marked version release. A tag is like a branch that doesn’t change. Unlike branches, tags, after being created, have no further history of commits.

Commands:-

1. git tag tagname
2. git tag -a v2.7   #annotated tags
3. git tag   #to list all the tags

Git Repository

A Git repository is a virtual storage of your project. It allows you to save versions of your code, which you can access when needed.

Commands:-

1. git init project_directory   #this will initialize git to specified directory
2. git clone repo_url   #to clone a repository in local
3. git remote add remote_name remote_repo_url   #remote repo url to your local git config, and set an upstream branch for your local branches.
4. git push -u remote_name local_branch_name   #map remote repository at to a ref in your local repo under
