What is Pytest?

PyTest is a testing framework that allows users to write test codes using Python programming language. It helps you to write simple and scalable test cases for databases, APIs, or UI. PyTest is mainly used for writing tests for APIs. It helps to write tests from simple unit tests to complex functional tests.

      - page1.html <- Information about git

      - page4.html <- Information about CI/CD
    - __init__.py <- It is used to mark directories on disk as Python package directories
  -static
    -css <- All the css files required for style the webpages are stored here
    -images <- All the images shown on the webpage are stored in this directory
    -js <- All the required javascript files used for functionality are stored here
  -templates <- the base html file which stores header and footer information is stored in this         for reuse and to keep the code DRY
    -run.py <- The run.py module is used to locate and run Python modules without          importing them first
  -tests <- All the test cases required to test various webpages are stored in this folder
  -.coveragerc <- This makes it easier to re-run coverage.py with consistent settings.
  -.dockerignore <- It is used to ignore files and folders when you try to build a Docker Image.
  -.gitignore <-Tells Git which files to ignore when committing your project to the GitHub          repository
  -.pylintrc <- Pylint is a tool that checks for errors in Python code, tries to enforce a coding         standard and looks for code smells
  -docker-compose.yml <- Orchestration file that contains the configuration to develop locally,         it overrides the Dockerfile to run the flask development server instead of running         the gunicorn server that is used for hosting on Heroku
  -Dockerfile <-File used to create an image to run in a container that runs the program
  -heroku.yml <- It is a manifest you can use to define your Heroku app. It allows you to build         docker images on heroku and specify addon and config vars to create app
  -readme.md <- It is used to provide orientation and guide on how to use your code for         engineers browsing code, especially first-time users.
  -requirements.txt <- It is used for specifying what python packages are required to run the         project you are looking at
  -setup.py <-This allows you to easily install Python packages.
